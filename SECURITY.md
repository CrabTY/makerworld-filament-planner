# Security Policy

## Supported versions

Security reports are accepted for the current `main` branch and the latest tagged release.

## Reporting a vulnerability or bug

Please report issues through GitHub Issues. If the repository enables GitHub's private vulnerability reporting later, use that path for security-sensitive reports.

Helpful details include:

- A clear description of the issue.
- Steps to reproduce.
- Affected commit, release, or deployment URL.
- Any relevant request/response examples with secrets removed.

## Scope

This project parses user-provided MakerWorld URLs, requests public profile metadata, and stores planning state in the user's browser. Reports involving input handling, request forwarding, dependency vulnerabilities, or exposed sensitive data are in scope.

Issues in third-party services, store pricing, inventory, shipping, or profile metadata accuracy should be reported to the relevant service owner.
